# Team & Roadmap

This page turns Orbit from a pure concept narrative into an execution narrative. Specific names can remain pseudonymous, but delivery commitments should be explicit.

## Team Snapshot

Current working assumption for public disclosure format:

- contributor count
- protocol engineering background summary
- smart contract and market design experience summary
- operations/security ownership model

This section should be updated with concrete numbers before broad external distribution.

## Security and Audit Plan

Planned progression:

1. Internal threat modeling and invariant review
2. Testnet deployment with public bug-bounty window
3. External smart contract audit round
4. Remediation and re-audit for high-severity findings

## Product Milestones

### Phase A: Spec Hardening

- finalize oracle mechanism and dispute parameters
- finalize initialization anti-manipulation design
- lock protocol-level risk disclosures

### Phase B: Testnet

- launch restricted testnet markets
- monitor liquidity behavior under real participation
- validate dispute and settlement operations end-to-end

### Phase C: Mainnet Rollout

- deploy audited contracts
- enable staged market categories
- activate referral and liquidity point incentives in guarded mode

## Governance Path

Initial operation is expected to be team-managed under transparent parameter controls. Governance decentralization can progress in stages:

1. publish parameter change policy
2. establish community signaling process
3. migrate selected controls to DAO governance after operational stability

## Delivery Transparency

Before each phase transition, Orbit should publish:

- what changed
- what was validated
- unresolved risks
- go/no-go criteria for next stage
